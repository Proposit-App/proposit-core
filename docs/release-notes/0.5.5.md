# 0.5.5

## New: Argument Parsing Module

AI-model-agnostic module for converting structured AI responses into fully populated `ArgumentEngine` instances with libraries. The library provides prompt construction, response schemas, and an ingestion pipeline — but does not call any AI model directly.

### Core parsing (`src/lib/parsing/`)

- **`ArgumentParser`** — subclassable class with two-phase API:
  - `validate(raw)` — validates AI response JSON against the (possibly extended) response schema
  - `build(response)` — constructs `ArgumentEngine`, `ClaimLibrary`, `SourceLibrary`, and `ClaimSourceLibrary` from the validated response
  - 6 protected mapping hooks (`mapArgument`, `mapClaim`, `mapSource`, `mapVariable`, `mapPremise`, `mapClaimSourceAssociation`) for injecting extension fields
- **`buildParsingPrompt(responseSchema, options?)`** — generates a system prompt for AI models with hand-crafted core instructions and auto-discovered extension field instructions from schema metadata
- **`buildParsingResponseSchema(options?)`** — merges developer extension schemas into the core response schema
- **`getParsingResponseSchema(schema?)`** — converts TypeBox schema to plain JSON Schema for AI model structured output configuration
- **Response schemas**: `TParsedClaim`, `TParsedVariable`, `TParsedSource`, `TParsedPremise`, `TParsedArgument`, `TParsedArgumentResponse` — all with `additionalProperties: true`

### Basics extension (`src/extensions/basics/`)

Demonstrates the extension pattern with common metadata fields:
- `TBasicsArgument` — adds `title` and optional `description`
- `TBasicsClaim` — adds `title` and `body`
- `TBasicsPremise` — adds `title`
- `BasicsArgumentParser` — subclass that maps these metadata fields during build
- `BasicsParsingSchema` — pre-built extended response schema

### Formula parser enhancement

`parseFormula()` now accepts word-based operators (`and`, `or`, `not`, `implies`, `iff`) alongside the existing symbolic operators (`&&`, `||`, `!`, `->`, `<->`, `∧`, `∨`, `¬`, `→`, `↔`).
