# 0.5.8

## Operator Nesting Restriction

Non-`not` operator expressions (`and`, `or`, `implies`, `iff`) can no longer be direct children of other operator expressions. A `formula` node must sit between them to make grouping explicit. The `not` operator is exempt as a child since its scope is unambiguous.

Enforced in `addExpression`, `insertExpression`, `wrapExpression`, and `removeExpression` (via pre-flight collapse simulation). Defense-in-depth guards in `removeAndPromote` and `collapseIfNeeded` prevent silent data corruption.

The CLI import pipeline (`buildExpressions`) and `toggleNegation` were updated to insert formula buffers automatically.

## Grammar Enforcement Config

A new `TGrammarConfig` system replaces the ad-hoc `skipNestingCheck` flag, providing configurable structural rule enforcement for expression trees.

- **`TGrammarOptions`** — per-rule toggles (`enforceFormulaBetweenOperators`)
- **`TGrammarConfig`** — extends `TGrammarOptions` with `autoNormalize` flag
- **`DEFAULT_GRAMMAR_CONFIG`** — all rules enforced, auto-normalize off (default behavior)
- **`PERMISSIVE_GRAMMAR_CONFIG`** — no enforcement (used by `fromData` during loading)

### Auto-normalization

When `autoNormalize` is `true`, `addExpression` auto-inserts formula buffer nodes instead of throwing. This is supported only in `addExpression` and bulk-loading paths. Compound operations (`insertExpression`, `wrapExpression`) and `removeExpression` always throw on violations regardless of this flag.

### Load-time enforcement control

`fromSnapshot`, `ArgumentEngine.fromSnapshot`, and `ArgumentEngine.fromData` accept an optional `grammarConfig` parameter controlling enforcement during loading only. The engine's ongoing config comes from the snapshot or constructor options.

- `fromData` defaults to `PERMISSIVE_GRAMMAR_CONFIG` during loading (preserving backward compatibility)
- `rollback` uses `PERMISSIVE_GRAMMAR_CONFIG` during loading

### Config threading

`grammarConfig` is added to `TLogicEngineOptions` and threads through `ArgumentEngine` -> `PremiseEngine` -> `ExpressionManager`. It round-trips through snapshots automatically. `PremiseEngine.snapshot()` now spreads `exprSnapshot.config` instead of cherry-picking fields.
