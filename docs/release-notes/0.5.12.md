# 0.5.12

## Bug Fixes

- **Snapshot checksumConfig serialization:** `snapshot()` at all levels (ArgumentEngine, PremiseEngine, ExpressionManager, VariableManager) now serializes `checksumConfig` `Set<string>` fields as `string[]` arrays. Previously, `JSON.stringify(snapshot())` would collapse Set fields to `{}`, losing all field names. The read-side normalization (`normalizeChecksumConfig`) remains in place as a safety net, but snapshots are now JSON-safe by default.

## New Exports

- **`serializeChecksumConfig(config)`** — Converts `Set<string>` fields to `string[]` arrays for JSON-safe serialization. Called automatically by `snapshot()` methods; exported for consumers who serialize configs independently.
