# 0.5.6

## LLM-Powered Argument Parsing

New `arguments parse` command extracts structured propositional arguments from natural-language text using an LLM.

- `arguments parse [text] --llm openai` — parse text into a persisted argument
- Supports positional argument or piped stdin input
- `--dry-run` flag outputs raw LLM JSON without persisting
- `--title` and `--description` flags for argument metadata
- `--model` flag to override the default model (gpt-5.4)
- API key resolved from `--api-key` flag or `OPENAI_API_KEY` env var

### LLM Provider Abstraction

- Generic `TLlmProvider<TResponse>` interface in `src/cli/llm/`
- `createLlmProvider(name, options)` factory with `resolveApiKey()` for env var fallback
- OpenAI provider uses native `fetch` with `json_schema` structured output

## Library Persistence

Claim, source, and claim-source association libraries are now persisted to disk as global JSON files (`~/.proposit-core/claims.json`, `sources.json`, `claim-source-associations.json`).

- `hydrateEngine` auto-loads libraries from disk when not explicitly provided
- Placeholder claim generation ensures backward compatibility with arguments created before library persistence
- `arguments parse` and `arguments import` merge new libraries into global state
- `importArgumentFromYaml` now returns libraries alongside the engine

## Claim and Source CLI Commands

New top-level `claims` and `sources` command groups for managing global libraries.

### Claims
- `claims list [--json]` — list all claims
- `claims show <claim_id> [--json]` — show all versions of a claim
- `claims add` — create a new claim
- `claims freeze <claim_id>` — freeze current version, create new mutable version

### Sources
- `sources list [--json]` — list all sources
- `sources show <source_id> [--json]` — show all versions of a source
- `sources add --text <text>` — create a new source
- `sources link-claim <source_id> <claim_id>` — create a claim-source association
- `sources unlink <association_id>` — remove a claim-source association
