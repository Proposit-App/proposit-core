# 0.6.2

## Improved sourceMiniIds guidance in parsing prompt

The LLM parsing prompt now includes explicit instructions to prevent a common
mistake where the model puts claim miniIds (e.g., "c1", "c2") into a claim's
`sourceMiniIds` field instead of source miniIds ("s1", "s2").

### Changes

- **New "Sources (External Citations)" section** in the system prompt — explains
  that sources are external citations (books, articles, URLs), not other claims,
  and that logical dependencies between claims should be expressed through
  premises and formulas.
- **Strict cross-type reference rules** in the MiniId Conventions section —
  `sourceMiniIds` accepts only `s`-prefixed IDs, `claimMiniId` accepts only
  `c`-prefixed IDs, `conclusionPremiseMiniId` accepts only `p`-prefixed IDs.
- **Schema description annotation** on `sourceMiniIds` — the JSON schema sent to
  the LLM now carries a `description` field reinforcing that the array must
  contain source references, not claim references.
- **Empty array guidance** — the prompt now instructs the LLM to leave
  `sourceMiniIds` as `[]` and the `sources` array empty when the input text has
  no external citations.
