# 0.5.7

## Basics Extension Integration

The CLI `arguments parse` command now uses the basics extension schemas (`BasicsParsingSchema` and `BasicsArgumentParser`), so the LLM is instructed to generate metadata for all entities:

- **Claims** — `title` (short summary) and `body` (detailed description)
- **Premises** — `title` (short label)
- **Arguments** — `title` (LLM-generated; `--title` flag overrides if provided)

The `--title` flag on `arguments parse` is now optional — when omitted, the LLM-generated title is used.

## Enhanced Claim Commands

- `claims add` accepts `--title <title>` and `--body <body>` options
- `claims update <claim_id> --title <title> --body <body>` — new command for updating claim metadata on mutable versions
- `claims list` displays claim title inline
- `claims show` displays title and body per version

## Enhanced Render Output

The `render` command now displays full argument metadata:

- Argument title and description header
- Premise titles alongside formula display strings
- Variable-to-claim title bindings (resolved from the claim library)
- Claim title and body in the claims section
- Source text in the sources section

## CLI Logging

All CLI invocations are logged to `~/.proposit-core/logs/cli.jsonl` as append-only JSONL. Each entry includes a timestamp and event name. The `arguments parse` command additionally logs:

- `parse:llm-response` — full LLM response with provider, model, input text, and raw JSON
- `parse:llm-error`, `parse:validation-error`, `parse:build-error` — error details on failure
